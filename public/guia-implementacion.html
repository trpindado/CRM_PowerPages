1. Requisitos previos

  • Cuenta de Azure — Puedes crear una cuenta gratuita en portal.azure.com (incluye 200 USD de credito)
  • Cuenta de Azure DevOps — Gratuita en dev.azure.com (hasta 5 usuarios, repos ilimitados)
  • Azure CLI — Herramienta de linea de comandos (version 2.50+)
  • Node.js 20 LTS — Runtime de la aplicacion
  • Git — Control de versiones

3. Instalar y configurar Azure CLI

Instalacion

Descarga e instala desde https://aka.ms/installazurecliwindows o via winget:

winget install --id Microsoft.AzureCLI

Login

az login
# Se abre el navegador para autenticarte con tu cuenta Microsoft

Instalar extension de Azure DevOps

az extension add --name azure-devops --yes

4. Configurar Azure DevOps

4.1 Crear organizacion y proyecto

  1. Accede a https://dev.azure.com e inicia sesion.
  2. Crea una organizacion (ej: teresariveropindado).
  3. Crea un proyecto llamado CRM_PowerPages con visibilidad Private y control de versiones Git.

Configurar defaults en CLI

az devops configure --defaults \
  organization=https://dev.azure.com/teresariveropindado \
  project=CRM_PowerPages

4.2 Subir codigo al repositorio

Inicializa Git en el directorio del proyecto y conecta con Azure DevOps:

# Inicializar repositorio local
git init
git remote add azure \
  https://teresariveropindado@dev.azure.com/teresariveropindado/CRM_PowerPages/_git/CRM_PowerPages

# Primer commit y push
git add -A
git commit -m "Initial commit - CRM PowerPages"
git push azure master --set-upstream
✔ El fichero .gitignore excluye automaticamente node_modules/, .env y crm.db del repositorio.

5. Crear Azure App Service

5.1 Resource Group

Agrupa todos los recursos de Azure en un mismo grupo logico:

az group create \
  --name CRM-PowerPages-RG \
  --location westeurope

5.2 App Service Plan

Crea el plan de hospedaje. El plan F1 es gratuito:

az appservice plan create \
  --name CRM-PowerPages-Plan \
  --resource-group CRM-PowerPages-RG \
  --sku F1 \
  --is-linux
ℹ El plan F1 es gratuito pero tiene limitaciones: 60 min CPU/dia, 1 GB RAM, sin dominio personalizado SSL. Para produccion usa B1 (Basic) o superior.

5.3 Crear la Web App

az webapp create \
  --resource-group CRM-PowerPages-RG \
  --plan CRM-PowerPages-Plan \
  --name crm-powerpages \
  --runtime "NODE:20-lts"

Esto crea la app accesible en https://crm-powerpages.azurewebsites.net

5.4 Variables de entorno

Configura las variables sensibles (como la API key de OpenAI) directamente en App Service, nunca en el codigo:

az webapp config appsettings set \
  --resource-group CRM-PowerPages-RG \
  --name crm-powerpages \
  --settings OPENAI_API_KEY="tu-clave-aqui"

# Habilitar npm install durante el despliegue
az webapp config appsettings set \
  --resource-group CRM-PowerPages-RG \
  --name crm-powerpages \
  --settings SCM_DO_BUILD_DURING_DEPLOYMENT=true
⚠ Nunca subas el fichero .env al repositorio. Las variables secretas se gestionan a traves de App Settings en Azure o mediante Azure Key Vault para mayor seguridad.

Alternativamente, puedes configurar las variables desde el portal de Azure DevOps en Pipelines → Library → Variable Groups para usarlas en los pipelines de CI/CD.

6. Despliegue de la aplicacion

Opcion A: Despliegue directo con CLI (recomendado para inicio)

El comando az webapp up empaqueta, sube y despliega en un solo paso:

az webapp up \
  --resource-group CRM-PowerPages-RG \
  --name crm-powerpages \
  --plan CRM-PowerPages-Plan \
  --runtime "NODE:20-lts" \
  --sku F1
ℹ Este comando crea un zip del directorio, lo sube a Kudu, ejecuta npm install automaticamente y arranca la app con npm start.

Opcion B: Despliegue via Pipeline CI/CD

Ver seccion 8. Pipeline CI/CD para despliegue automatico en cada push.

Verificar el despliegue

# Ver logs en tiempo real
az webapp log tail \
  --resource-group CRM-PowerPages-RG \
  --name crm-powerpages

# Reiniciar la app si es necesario
az webapp restart \
  --resource-group CRM-PowerPages-RG \
  --name crm-powerpages

7. Autenticacion con Azure AD (EasyAuth)

Azure App Service incluye un modulo de autenticacion integrado (EasyAuth) que protege la aplicacion sin modificar el codigo. Cualquier usuario no autenticado es redirigido al login de Microsoft.

7.1 Registrar la aplicacion en Azure AD

# Crear el registro de aplicacion
az ad app create \
  --display-name "CRM PowerPages" \
  --web-redirect-uris "https://crm-powerpages.azurewebsites.net/.auth/login/aad/callback" \
  --sign-in-audience "AzureADandPersonalMicrosoftAccount"

# Habilitar emision de ID tokens
az ad app update \
  --id <APP_ID> \
  --enable-id-token-issuance true

# Crear un client secret
az ad app credential reset \
  --id <APP_ID>
⚠ Guarda el client secret que devuelve el ultimo comando. No se puede recuperar despues.

7.2 Configurar EasyAuth en App Service

az webapp auth update \
  --resource-group CRM-PowerPages-RG \
  --name crm-powerpages \
  --enabled true \
  --action LoginWithAzureActiveDirectory \
  --aad-client-id "<APP_ID>" \
  --aad-client-secret "<CLIENT_SECRET>" \
  --aad-token-issuer-url "https://login.microsoftonline.com/common/v2.0" \
  --aad-allowed-token-audiences "https://crm-powerpages.azurewebsites.net"

7.3 Restringir acceso a usuarios especificos

Por defecto, cualquier cuenta Microsoft puede autenticarse. Para restringir el acceso a un grupo de usuarios:

  1. Ve a Azure Portal → Azure Active Directory → Aplicaciones empresariales.
  2. Busca la aplicacion CRM PowerPages.
  3. En Propiedades, activa "Asignacion de usuarios requerida" = Si.
  4. En Usuarios y grupos, agrega solo los usuarios o grupos que deben tener acceso.

Tambien se puede hacer via CLI:

# Requerir asignacion de usuarios
az ad sp update \
  --id <APP_ID> \
  --set appRoleAssignmentRequired=true

8. Pipeline CI/CD (azure-pipelines.yml)

El fichero azure-pipelines.yml en la raiz del proyecto define el pipeline de integracion y despliegue continuo:

# azure-pipelines.yml
trigger:
  - master

pool:
  vmImage: 'ubuntu-latest'

steps:
  - task: NodeTool@0
    inputs:
      versionSpec: '20.x'

  - script: npm install

  - task: ArchiveFiles@2
    inputs:
      rootFolderOrFile: '$(System.DefaultWorkingDirectory)'
      archiveType: 'zip'

  - task: AzureWebApp@1
    inputs:
      azureSubscription: 'Azure subscription 1'
      appName: 'crm-powerpages'
      runtimeStack: 'NODE|20-lts'

Activar el pipeline

  1. En Azure DevOps, ve a Pipelines en el menu lateral.
  2. Haz clic en "Create Pipeline".
  3. Selecciona "Azure Repos Git" → "CRM_PowerPages".
  4. Detectara el azure-pipelines.yml automaticamente. Haz clic en "Run".
  5. La primera vez te pedira autorizar la Service Connection con tu suscripcion Azure.

Crear la Service Connection

  1. Ve a Project Settings → Service connections.
  2. New service connection → Azure Resource Manager → Service principal (automatic).
  3. Selecciona tu suscripcion y resource group CRM-PowerPages-RG.
  4. Nombra la conexion exactamente: Azure subscription 1.
  5. Marca "Grant access permission to all pipelines".
⚠ Las cuentas nuevas de Azure DevOps requieren solicitar el free parallelism grant para usar agentes hospedados. Rellena el formulario en https://aka.ms/azpipelines-parallelism-request (tarda 2-3 dias laborables). Mientras tanto, usa az webapp up para desplegar.
