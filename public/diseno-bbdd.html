1. Visión general

El CRM GNL utiliza SQLite (better-sqlite3) como motor de base de datos embebido. El fichero de base de datos es crm.db en la raíz del proyecto.

CaracterísticaValor
MotorSQLite 3 (via better-sqlite3)
Ficherocrm.db (raíz del proyecto)
Journal modeWAL (Write-Ahead Logging)
Foreign keysON (PRAGMA foreign_keys = ON)
Número de tablas6 (5 de negocio + Usuarios)
Archivos auxiliarescrm.db-wal, crm.db-shm
ℹ Los ficheros crm.db-wal y crm.db-shm son archivos de WAL activos. Al copiar la BD entre servidores, deben copiarse los tres ficheros o bien detener el servidor antes de copiar.

3. Tabla Pais

Catálogo de países y territorios relevantes para el negocio GNL. Es la tabla padre de Entidades.

CampoTipoRestriccionesDescripción
CodigoPaisNormalizado PK TEXT PRIMARY KEY Código ISO 3166-1 alpha-3 (3 letras mayúsculas). Calculado automáticamente en el alta de países. Excepción: LNG para el territorio virtual de GNL.
NombreTEXT—Nombre del país en español
RegionTEXT—Región geográfica o comercial (p.ej. Asia-Pacífico, Europa…)
ReferenciaIndiceTEXT—Referencia interna de índice de precios o mercado
LinkFichaPaisTEXT—URL al informe de riesgo CESCE. Se muestra como icono 🌐 en la tabla. Precargado para ~49 países.
PersonaReferenciaOportunTEXT—Persona de referencia interna de Naturgy para ese mercado
ComentariosTEXT—Notas libres

4. Tabla Entidades

Contrapartes comerciales (empresas, organizaciones). Tabla central del CRM; referenciada por Contactos, Oportunidades y Documentos.

CampoTipoRestriccionesDescripción
CodigoEntidad PK TEXT PRIMARY KEY Identificador único. Formato libre; se puede generar automáticamente o asignar manualmente al importar.
Compania NN TEXT NOT NULL Nombre de la empresa
RegionTEXT—Región de negocio
TipoTEXT—Clasificación: Buyer, Seller, Trader, Shipper, Broker…
CodigoPaisNormalizado FK TEXT REFERENCES Pais(CodigoPaisNormalizado) País de la entidad. Selector desplegable en el formulario de alta/edición.
FiscalCodeTEXT—CIF, NIF u otro código fiscal local
LEITEXT—Legal Entity Identifier (ISO 17442, 20 caracteres)
TickerTEXT—Símbolo bursátil
DunsNumberTEXT—D-U-N-S Number (Dun & Bradstreet)
DireccionTEXT—Dirección postal completa
ComentariosTEXT—Notas libres

NombreTEXT—Nombre completo del contacto
CargoTEXT—Título o posición profesional
EmailTEXT—Correo electrónico principal
Telefono1TEXT—Teléfono principal
Telefono2TEXT—Teléfono alternativo
ViaTEXT—Canal preferido de comunicación
FechaUltimoContactoTEXT—Fecha del último contacto (formato libre)
DemorarContactoAfechaTEXT—Fecha de próximo contacto planificado

7. Tabla Documentos

Estado de la documentación legal requerida para operar con cada entidad: KYC (Know Your Customer), NDA (Non-Disclosure Agreement) y MSPA (Master Sale & Purchase Agreement).

CampoTipoRestriccionesDescripción
id PK AI INTEGER PRIMARY KEY AUTOINCREMENT Identificador numérico autoincremental
CodigoEntidad FK NN TEXT NOT NULL REFERENCES Entidades(CodigoEntidad) Entidad a la que pertenece el registro de documentación
KYC_S_NTEXT—Estado del KYC: S (completado), N (pendiente) u otro valor
KYC_linkTEXT—URL al documento/expediente KYC. Se muestra como icono 📎 junto a KYC_S_N.
NDA_S_NTEXT—Estado del NDA: S (firmado), N (pendiente) u otro valor
FechaExpiracionNDATEXT—Fecha de expiración del NDA (formato libre)
NDALinkTEXT—URL al documento NDA. Se muestra como icono 📎 junto a NDA_S_N.
MSPASNTEXT—Estado del MSPA: S (en vigor), N (pendiente) u otro valor
LinkMSPATEXT—URL al documento MSPA. Se muestra como icono 📎 junto a MSPASN.
ComentariosTEXT—Notas libres
ℹ Los pares de campos (estado + enlace) se gestionan a través de la propiedad linkPairs del esquema frontend, que permite renderizar el icono de enlace al lado del campo de estado en la tabla.

8. Tabla Usuarios

Tabla de autenticación y autorización. Independiente del resto del modelo de negocio.

CampoTipoRestriccionesDescripción
id PK AI INTEGER PRIMARY KEY AUTOINCREMENT Identificador numérico autoincremental
username NN TEXT UNIQUE NOT NULL Nombre de usuario para el login
password NN TEXT NOT NULL Contraseña en texto plano (se recomienda cambiar las contraseñas por defecto)
nombreTEXT—Nombre completo del usuario
rol NN TEXT NOT NULL DEFAULT 'user' Rol de acceso: admin o comercial

10. Configuración SQLite

PRAGMAValorMotivo
journal_mode WAL Permite lectura concurrente mientras se escriben datos. Necesario para entornos con múltiples peticiones simultáneas.
foreign_keys ON Activa la comprobación de integridad referencial a nivel de motor.

11. Sistema de migraciones

Las migraciones se ejecutan en la función runMigrations() de database.js al arrancar el servidor. Son idempotentes: comprueban el estado actual antes de aplicar cambios.

Migraciones aplicadas

DescripciónCondición de aplicación
Precarga de enlaces CESCE para 49 países LinkFichaPais IS NULL OR LinkFichaPais != ? para cada país
Corrección de datos de países sin código normalizado Se aplica solo si el campo está vacío
ℹ Para añadir una nueva migración, agrégala al bloque de runMigrations() en database.js con su propia comprobación de idempotencia.

Exportación de datos (Migracion/)

El script Migracion/exportar_sqlite.js genera 5 ficheros CSV en Migracion/exports/ con todos los datos de la BD:

FicheroTabla
01_Pais.csvPais
02_Entidades.csvEntidades
03_Contactos.csvContactos
04_Oportunidades.csvOportunidades
05_Documentos.csvDocumentos

Estos ficheros no se sincronizan con GitHub (están en .gitignore). Se regeneran con:

node Migracion/exportar_sqlite.js

12. Reglas de codificación

Código de país (CodigoPaisNormalizado)

Sigue el estándar ISO 3166-1 alpha-3: 3 letras mayúsculas en inglés. La función calcAutoCode(nombre) del frontend contiene un mapa de ~200 países en español a su código ISO3.

Excepción: LNG — territorio virtual que agrupa oportunidades sin localización geográfica específica.

Código de entidad (CodigoEntidad)

Formato libre. Por convención, se suele usar un código alfanumérico descriptivo. Ejemplos: SHELL-GNL, QATAR-ENERGY-01.

Códigos autogenerados

Si el campo de código no se proporciona al crear un registro, el servidor lo genera con db.generateNextCode(table, codeField, prefix).
